0) Final Architecture (n8n-native)
	•	Intake & Filtering: Apify → n8n (cron/webhook) → Airtable (upsert).
	•	Offer Generation: Airtable button (→ n8n webhook) → validations (Apify/HTTP + rules) → Google Drive “copy from template” → Google Docs “replace placeholders” → Google Drive “export to PDF” → Airtable (attach PDF) → notifications.
	•	Addenda Logic (rules):
	•	Lead-Based Paint addendum if YearBuilt < 1978.
	•	Tenant Addendum if rented = true or MLS shows tenant remarks.
	•	Closing Date: shift to next business day if weekend/US holiday.
	•	Status Tracking: Airtable single source of truth; n8n updates.

3) n8n Workflow A — “Property Intake & Filter” (replaces Make.com Property Check)

Triggering: either on a schedule (Cron) or Apify webhook.

Node map (names + exact key settings)

[A01] Cron
	•	Mode: Every X minutes/hours (choose)
	•	Timezone: your local

[A02] Airtable → List Records (Sources)
	•	Base: Real Estate Offers
	•	Table: Sources
	•	Filter: Enabled = 1

[A03] Split In Batches
	•	Batch size: 1

[A04] Apify → Run Actor
	•	Actor: apify/website-scraper (or your custom Playwright actor)
	•	Input JSON (typical minimal):

{
  "startUrls": [ {"url":"{{$json["Start URL"]}}"} ],
  "maxRequestsPerCrawl": 100,
  "linkSelector": "a",
  "pageFunction": "async ({ request, $, log }) => { /* extract MLS cards to return */ }"
}


	•	Poll for finish: Enabled

[A05] Function (Filter & Normalize)
	•	Code (example):

const crit = {
  city: ['Dallas','Plano','Frisco'],     // your target cities
  minPrice: 150000,
  maxPrice: 650000,
  minBeds: 2,
  minBaths: 1
};
const items = $json.items || [];
const out = [];
for (const it of items) {
  // expect it to contain: address, city, price, beds, baths, yearBuilt, url, mlsId
  if (!it.address || !it.city || !it.price) continue;
  if (!crit.city.includes(it.city)) continue;
  if (it.price < crit.minPrice || it.price > crit.maxPrice) continue;
  if ((it.beds ?? 0) < crit.minBeds) continue;
  if ((it.baths ?? 0) < crit.minBaths) continue;
  out.push(it);
}
return out.map(x => ({ json: x }));

4) n8n Workflow B — “Offer Generation & Validation” (replaces Make.com + MultiOn + Parsio)

Trigger: Airtable button hits this webhook with recordId.

Node map (names + exact key settings)

[B01] Webhook (POST/GET)
	•	Path: /offer/generate
	•	Respond: Immediately with 200 JSON { "ok": true }
	•	Capture: recordId from querystring/body.

[B02] Airtable → Read (Offers)
	•	Base/Table: Real Estate Offers / Offers
	•	Operation: Get Record
	•	Record ID: {{$json.query.recordId || $json.body.recordId}}

[B03] Function (Compute Financials)
	•	Code:

const offer = $json;
const pct = Number(offer["Down Payment %"] ?? 0.35) || 0.35;
const amount = Number(offer["Offer Amount (USD)"] ?? 0);
const down = Math.round(amount * pct * 100) / 100;
const remainder = Math.round((amount - down) * 100) / 100;
return [{
  json: {
    downPct: Math.round(pct*100),
    downUsd: down,
    remainderUsd: remainder
  }
}];



[B04] Airtable → Update (Offers, set “Validating”)
	•	Record: {{$json["id"]}} (from B02)
	•	Fields:
	•	Status ← Validating
	•	Down Payment % ← {{$json("B03").downPct / 100}}
	•	Down Payment (USD) ← {{$json("B03").downUsd}}
	•	Remainder (USD) ← {{$json("B03").remainderUsd}}

[B05] HTTP Request (Holiday API: Nager.Date)
	•	Method: GET
	•	URL: https://date.nager.at/api/v3/PublicHolidays/{{$now.format("YYYY")}}/US
	•	Response: JSON
	•	Notes: If non-US, replace US accordingly.

[B06] Function (Closing Date Adjust)
	•	Code:

const rec = $items("B02")[0].json;
const holidays = $items("B05")[0].json || [];
const proposed = new Date(rec["Proposed Closing Date"]);
if (isNaN(proposed)) {
  return [{ json: { adjusted: null, note: "No proposed closing date" } }];
}
// weekend adjust
const wd = proposed.getDay(); // 0 Sun, 6 Sat
if (wd === 6) proposed.setDate(proposed.getDate() + 2);
if (wd === 0) proposed.setDate(proposed.getDate() + 1);
// holiday adjust
const isHoliday = (d) => {
  const ymd = d.toISOString().slice(0,10);
  return holidays.some(h => h.date === ymd);
};
while (isHoliday(proposed) || proposed.getDay() === 0 || proposed.getDay() === 6) {
  proposed.setDate(proposed.getDate() + 1);
}
return [{
  json: {
    adjusted: proposed.toISOString().slice(0,10)
  }
}];



[B07] IF (Add Lead-Based Paint Addendum)

[B08] IF (Add Tenant Addendum)

const v = $json || {};
const flags = [];
if (v.addrMatch === false) flags.push("Address mismatch");

    validationFlags: flags.join("; ") || "OK",
    validationErrors: flags.length ? flags.join("; ") : ""
  }
}];

7) Validation Rules (authoritative)
	•	Lead-Based Paint addendum: Year Built < 1978 ⇒ add to Addenda Required.
	•	Tenant addendum: Is Rented = true OR tenantRemarkFound = true.
	•	Closing Date: If Saturday/Sunday or US public holiday, move to next business day.
	•	Title Company present: If actor fails to find mention in MLS page, flag in Validation Flags (doesn’t block generation; prompts human review).

8) Error-Handling & Rate Limiting
	•	For all Apify and HTTP nodes: Retry On Fail: 3, Delay: 2s backoff.
	•	Centralize failures in Error Workflow:
	•	Update Status = Error
	•	Write Validation Errors with node name + message
	•	Slack DM (optional)
