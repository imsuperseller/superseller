DESIGN VARIATIONS GENERATOR
Generate n = 3 files

Instructions
Look at the design that has been attached.
Create 3 different versions of that same design. Make each one look different but keep the same basic idea and functionality.
Follow any additional instructions given in the prompt.

File Management
- Check if a "variations" folder exists
- If it exists:
  • Rename the reference design to "source.html"
  • Delete any other files in the folder
  • If "source.html" already exists, delete the old one first
- If no "variations" folder exists:
  • Create the "variations" folder
  • Put the reference design in as "source.html"

Create Files
Make these files in the variations folder:
- design1.html
- design2.html
- design3.html
- Continue to design{n}.html if more than 3 requested

Each design should be completely different visually but work the same way.
That's it. Keep it simple. Make them look different. Follow any extra instructions given.

VARIATION TYPES
Create distinctly different visual approaches:

1. **Color Scheme Variation** (design1.html)
   - Change the entire color palette
   - Maintain the same layout and functionality
   - Use complementary or contrasting colors
   - Consider dark/light theme variations

2. **Layout & Structure Variation** (design2.html)
   - Reorganize the layout while keeping functionality
   - Change grid systems, spacing, or component arrangement
   - Modify navigation or content flow
   - Keep the same interactive elements

3. **Style & Aesthetic Variation** (design3.html)
   - Change visual style (modern, retro, futuristic, etc.)
   - Modify typography, shadows, borders, or effects
   - Adjust visual hierarchy and emphasis
   - Maintain usability and accessibility

ITERATION GUIDELINES
- Preserve all functionality from the source design
- Maintain accessibility standards
- Keep responsive design principles
- Ensure cross-browser compatibility
- Optimize for performance
- Include proper semantic HTML structure

QUALITY REQUIREMENTS
- Each variation should be production-ready
- Include proper meta tags and viewport settings
- Use semantic HTML elements
- Implement responsive design
- Ensure accessibility compliance
- Optimize for loading speed

OUTPUT STRUCTURE
```
variations/
├── source.html        # Original reference design
├── design1.html       # Color scheme variation
├── design2.html       # Layout & structure variation
└── design3.html       # Style & aesthetic variation
```

EVALUATION CRITERIA
- Visual distinctiveness between variations
- Functional consistency across all versions
- Accessibility and usability standards
- Performance and optimization
- Cross-browser compatibility
- Mobile responsiveness
