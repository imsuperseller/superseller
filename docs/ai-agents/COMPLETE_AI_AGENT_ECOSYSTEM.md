## 🔄 **AGENT INTEGRATION**

### **Shared Infrastructure**
All three agents share common infrastructure components:

```javascript
// Shared configuration
const sharedConfig = {
  openai: {
    apiKey: process.env.OPENAI_API_KEY,
    model: "gpt-4"
  },
  database: {
    mongodb: process.env.MONGODB_URI,
    postgres: process.env.POSTGRES_URI
  },
  n8n: {
    baseUrl: process.env.N8N_BASE_URL,
    apiKey: process.env.N8N_API_KEY
  },
  notifications: {
    email: process.env.SMTP_CONFIG,
    slack: process.env.SLACK_WEBHOOK
  }
};
```

### **Data Flow Between Agents**
```javascript
// Example: Data flow from onboarding to customer success
const onboardingToSuccess = async (customerSlug) => {
  // 1. Onboarding agent creates customer profile
  const onboardingData = await runOnboardingAgent(customerSlug);
  
  // 2. Customer success agent starts monitoring
  await runCustomerSuccessAgent(customerSlug, onboardingData);
  
  // 3. System monitoring agent ensures infrastructure support
  await runSystemMonitoringAgent();
  
  return { onboardingData, successData, monitoringData };
};
```

### **Error Handling & Recovery**
```javascript
// Shared error handling across all agents
const handleAgentError = async (error, agent, context) => {
  console.error(`❌ Error in ${agent} - ${context}:`, error.message);
  
  // Log error for analysis
  await logError(error, { agent, context });
  
  // Attempt recovery based on error type
  if (error.code === 'RATE_LIMIT') {
    await delay(5000);
    return await retryOperation(agent, context);
  }
  
  // Notify admin for critical errors
  if (error.critical) {
    await notifyAdmin(error, { agent, context });
  }
  
  throw error;
};
```
