## 🔍 Validation Tool Issue Analysis

### Current Status

**Error**: `"no such column: node_type"` when validating workflows  
**Tool**: n8n-mcp validation tool (`n8n_validate_workflow`)  
**n8n Version**: 1.119.1 ✅  
**MCP Tool Version**: 2.22.15 ✅

### Root Cause

The validation tool (`n8n-mcp`) is querying a `node_type` column that **does not exist** in n8n's database schema:

- ❌ Validation tool expects: `node_type` column
- ✅ n8n database has: `installed_nodes` table (different structure)
- ✅ n8n database has: `workflow_entity` table (stores nodes as JSON)

### Impact

- **Workflow Execution**: ✅ **NOT AFFECTED** - Workflows execute correctly
- **Validation Tool**: ❌ **BROKEN** - Shows false errors for all nodes
- **Workflow Data**: ✅ **CORRECT** - All data stored properly in database

## 📊 Database Schema (n8n 1.119.1)

### Actual Schema

**Tables**:
- `workflow_entity` - Stores workflows (nodes stored as JSON in `nodes` column)
- `installed_nodes` - Stores installed node packages
- `migrations` - Database migration tracking
- `settings` - n8n settings
- `credentials_entity` - Encrypted credentials
- `execution_entity` - Workflow executions

**No `node_type` Column**: The validation tool is querying a column that doesn't exist.

### Node Storage

Nodes are stored as **JSON** in the `workflow_entity.nodes` column:
```json
{
  "id": "a0a586a6-589f-4873-9ada-ed196527c8ce",
  "name": "Read Products CSV",
  "type": "n8n-nodes-base.readBinaryFile",
  "typeVersion": 1,
  "parameters": {
    "fileName": "/home/node/.n8n/data/products.csv"
  }
}
```

## 🔒 Security Notes

**Command Injection Vulnerability** (n8n 1.119.0 and earlier):
- **Affected**: Execute Command node
- **Mitigation**: Disable node via `NODES_EXCLUDE='["n8n-nodes-base.executeCommand"]'`
- **Status**: Not fixed in 1.119.1 (patch release only)
