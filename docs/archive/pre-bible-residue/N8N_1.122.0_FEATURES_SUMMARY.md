## 🎯 Key Features in 1.122.0

### **1. Native MCP Server HTTP Endpoint** ⚠️ **DOES NOT WORK**

**What It Is**: n8n 1.122.0+ includes a built-in MCP (Model Context Protocol) server that can be accessed via HTTP endpoint, but **this endpoint returns 404 errors** and does not work.

**⚠️ Current Working Solution**: Use npx mode instead (see `docs/infrastructure/MCP_CONFIGURATION.md`)

**⚠️ Status**: ❌ **DOES NOT WORK** - HTTP endpoint returns 404 errors

**Note**: These benefits were expected but the endpoint does not work:
- ❌ **No Local Wrapper Needed**: Would eliminate wrapper scripts (but endpoint doesn't work)
- ❌ **Simpler Configuration**: Would only need URL and JWT token (but endpoint doesn't work)
- ❌ **Better Performance**: Would use HTTP/SSE instead of stdio (but endpoint doesn't work)
- ❌ **Native Feature**: Built into n8n 1.122.0+ but returns 404 (does not work)

**⚠️ Endpoint Details** (DOES NOT WORK):
- ❌ **HTTP endpoint returns 404** - Endpoint does not exist
- ❌ **SSE connection fails** - Cannot connect
- **Current Solution**: Use npx mode instead (see `docs/infrastructure/MCP_CONFIGURATION.md`)

**Current Status**:
- ⚠️ **Rensto VPS**: HTTP endpoint returns 404 - Use npx mode instead (see `docs/infrastructure/MCP_CONFIGURATION.md`)
- ❌ **Tax4Us Cloud**: HTTP endpoint returns 404 - Use npx mode instead (see `docs/infrastructure/MCP_CONFIGURATION.md`)
- ✅ **Shelly Cloud**: Using npx mode (HTTP endpoint returns 404 for all instances)

**⚠️ Working Configuration** (npx mode - see `docs/infrastructure/MCP_CONFIGURATION.md`):
```json
{
  "mcpServers": {
    "n8n-rensto": {
      "command": "npx",
      "args": ["-y", "n8n-mcp"],
      "env": {
        "N8N_API_URL": "http://172.245.56.50:5678",
        "N8N_API_KEY": "[API_KEY]",
        "MCP_MODE": "stdio",
        "LOG_LEVEL": "error"
      }
    }
  }
}
```

**⚠️ Note**: HTTP endpoint returns 404. See `docs/infrastructure/MCP_CONFIGURATION.md` for working npx mode solution.
