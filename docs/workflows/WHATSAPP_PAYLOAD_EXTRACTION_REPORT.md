## 🐛 Critical Issue Found

### Issue #1: PDF Misidentification

**Severity**: 🔴 HIGH  
**Type**: Misidentification  
**Execution**: 20159  
**Message ID**: `3B60EE6E7C1463A09C51`

**Problem**:
- User sent a PDF file (`Invoice-633F2A51-0023.pdf`) with caption "analyze"
- Smart Message Router identified it as `messageType: "text"` instead of `"document"`
- Router extracted the caption as text content but ignored the PDF

**Evidence**:
```json
{
  "payload": {
    "hasMedia": true,
    "media": {
      "mimetype": "application/pdf",
      "filename": "Invoice-633F2A51-0023.pdf"
    },
    "_data": {
      "message": {
        "documentWithCaptionMessage": {
          "message": {
            "documentMessage": {
              "caption": "analyze"
            }
          }
        }
      }
    }
  }
}
```

**Router Output**:
```json
{
  "messageType": "text",  // ❌ WRONG - should be "document"
  "textContent": "analyze",
  "mediaUrl": "http://173.254.201.134:3000/api/files/rensto-whatsapp/3B60EE6E7C1463A09C51.pdf"
}
```

**Root Cause**:
Smart Message Router checks for `documentMessage` but when there's a `documentWithCaptionMessage`, it falls through to text detection because the caption is treated as body text.

**Fix Required**:
Update Smart Message Router to check for `documentWithCaptionMessage` before checking for text content:

```javascript
// Current (BROKEN):
if (body || conversation) {
  messageType = 'text';
}

// Fixed:
const hasDocument = _dataMessage.documentMessage || 
                   _dataMessage.documentWithCaptionMessage;
const hasCaptionDoc = _dataMessage.documentWithCaptionMessage;

if (hasDocument || hasCaptionDoc) {
  messageType = 'document';
  if (hasCaptionDoc) {
    caption = hasCaptionDoc.message?.documentMessage?.caption || 
             hasCaptionDoc.caption || 
             body;
  }
} else if (body || conversation) {
  messageType = 'text';
}
```
