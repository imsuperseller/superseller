**Part B — API route for Stripe session creation:**

Create `apps/web/superseller-site/src/app/api/checkout/create-session/route.ts`:

POST endpoint that:
1. Accepts `{ productSlug, email, phone, businessName }` in body
2. Looks up product config via getProductConfig(productSlug)
3. Creates a Stripe Checkout Session with:
   - mode: 'subscription'
   - customer_email: email
   - metadata: { phone, businessName, productName: product.name, serviceType: product.serviceType }
   - success_url: `${NEXT_PUBLIC_SITE_URL}/checkout/success?session_id={CHECKOUT_SESSION_ID}`
   - cancel_url: `${NEXT_PUBLIC_SITE_URL}/checkout/${productSlug}`
   - line_items with the Stripe price ID from product config
4. Returns `{ sessionUrl: session.url }` for client-side redirect

Use Stripe SDK (already in package.json). Handle missing price ID gracefully (return 400).
