2. Add a new exported query function `getPendingAdminApprovalRequest`:
```typescript
export async function getPendingAdminApprovalRequest(): Promise<ChangeRequestRow | null> {
    const rows = await query<any>(
        `SELECT id, group_id, tenant_id, message_body, intent, scope, scene_number,
                change_summary, status, estimated_cost_cents, poll_message_id,
                character_bible_version_id, admin_approval_poll_id, created_at, updated_at
         FROM change_requests
         WHERE status = 'pending-admin-approval'
         ORDER BY created_at DESC
         LIMIT 1`,
    );
    return rows[0] ? mapChangeRequestRow(rows[0]) : null;
}
```
Note: This queries globally (not per groupId) because there's only one admin and the concurrency guard ensures at most one pending-admin-approval at a time.

**1. `classifyChangeDelta(changeDelta: Record<string, any>)`**

- Empty changeDelta => `hasVisualChanges: false`, `affectedSceneCount: 0`, `changedFields: []`

**2. `sendAdminCharacterChangeReview(params)`**
Params: `{ groupId, tenantId, changeRequestId, changeDelta, customerName, affectedSceneCount, costDollars }`
- Resolve admin chatId: `phoneToChatId(config.admin.defaultPhone)`
- Send text message with formatted diff:
  ```
  --- Character Change Request ---

  Customer: {customerName}
  Change request ID: {changeRequestId}

  Changed fields:
  * {field}: "{from}" -> "{to}"

  Affected scenes: {affectedSceneCount}
  Estimated cost: ${costDollars}

  Please review and respond with your decision:
  ```
- Send poll to adminChatId: `"Approve character change for {customerName}?"` with options: `["Approve all {N} scenes", "Deny", "Select specific scenes"]`
- Update change_requests: set `status = 'pending-admin-approval'`, set `admin_approval_poll_id = pollMessageId`
- Return the poll message ID (or null on failure)

**3. `handleAdminCharacterApprovalPollVote(params)`**
Params: `{ pollOption: string, adminChatId: string }`
- Query `getPendingAdminApprovalRequest()` (from change-request-intake.ts)
- If no pending request, return `false`
- Parse pollOption:
  - Starts with "Approve all": update status to `admin-approved`, dispatch `characterRegenQueue.add("regen-character", { changeRequestId, sceneIndex: 0, tenantId, groupId, characterBibleId: cr.character_bible_version_id, affectedSceneIndices: [0,1,2,3,4] })`, send ack to customer group "Admin approved your character change! Regenerating {N} scenes...", return `true`
  - Equals "Deny": update status to `admin-denied`, send to customer group "Admin reviewed and declined this change. You can describe a different change if you'd like.", return `true`
  - Equals "Select specific scenes": send text to adminChatId "Which scenes should be regenerated? Reply with scene numbers (1-5), separated by commas. Example: 1, 3, 5", set a `pending_scene_selection` flag — store in change_requests by updating status to `pending-scene-selection`, return `true`
  - No match: return `false`

**4. `handleAdminSceneSelectionText(params)`**
Params: `{ messageBody: string, adminChatId: string }`
- Query for change request with status `pending-scene-selection`
- Parse scene numbers from messageBody using regex `/\d+/g`, filter to 1-5 range, convert to 0-based indices
- If valid scene numbers found: update status to `admin-approved`, dispatch `characterRegenQueue.add("regen-character", { changeRequestId, sceneIndex: indices[0], tenantId, groupId, characterBibleId, affectedSceneIndices: indices })`, send ack to customer group, return `true`
- If no valid numbers: send "Could not parse scene numbers. Please reply with numbers 1-5 separated by commas." to adminChatId, return `true` (handled but retrying)
- If no pending request: return `false`

**5. `handleNameOnlyChange(params)`**
Params: `{ groupId, changeRequestId, revealUrl, newName }`
- Send video to groupId with caption: `"Meet {newName} -- your updated AI character!"`
- Update change request status to `completed` with `{ estimatedCostCents: 0 }`
- Return void
