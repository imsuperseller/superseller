## What Was Built

### shot-types.ts
- `ShotType` discriminated union: `dialogue | narrative | environment | product | social | music`
- `BudgetTier` union: `budget | standard | premium`
- `BUDGET_CEILINGS`: `{ budget: 0.05, standard: 0.12, premium: 999 }`
- `SHOT_DEFAULT_MODELS`: maps each shot type to `{ provider, modelId, useCase }`
- `SHOT_TYPE_LABELS`: human-readable labels for each shot type
- `ShotRequest` interface with required fields (shotType, budgetTier, prompt) + optional fields

### provider-adapters/types.ts
- `ProviderAdapter` interface: `submitJob / pollStatus / cancelJob / estimateCost`
- `AdapterJobResult`: `{ externalJobId, provider }`
- `AdapterPollResult`: `{ status, resultUrl?, error? }`

### provider-adapters/kie-adapter.ts
- Wraps `createKlingTask` and `getTaskStatus` from `kie.ts` ‚Äî no raw Kie.ai fetch
- Video shots: delegates to `createKlingTask` with `mode: 'pro'`
- Music shots: calls Kie.ai Suno endpoint directly
- `pollStatus` maps KieTaskResponse status + extracts `video_url` or `resultUrls[0]`
- `cancelJob` is a no-op with warning (kie.ts has no cancel API)
- `estimateCost`: `costPerUnit * (duration / 5)` for video, flat `costPerUnit` for music

### provider-adapters/fal-adapter.ts
- Uses native `fetch` to `https://queue.fal.run/{modelId}` ‚Äî no SDK dependency
- Auth: `Key {FAL_API_KEY}` Bearer header (fal.ai uses `Key` prefix, not `Bearer`)
- `externalJobId` encoded as `{modelId}::{requestId}` for stateless poll/cancel routing
- Status mapping: `IN_QUEUE ‚Üí pending, IN_PROGRESS ‚Üí processing, COMPLETED ‚Üí completed, FAILED ‚Üí failed`
- Extracts result URL from `output.video.url` or `output.video_url`
- Graceful startup: logs warning if `FAL_API_KEY` missing, does not throw in constructor
