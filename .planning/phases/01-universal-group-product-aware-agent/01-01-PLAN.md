Export interface `ProductInfo`:
```typescript
export interface ProductInfo {
  productName: string;
  productId: string | null;
  status: string;
  type: string | null; // ServiceInstance.type or Subscription.subscriptionType
}
```

Export `async function fetchTenantProducts(tenantId: string): Promise<ProductInfo[]>`:
- Query ServiceInstance: `SELECT "productName", "productId", status, type FROM "ServiceInstance" WHERE "tenantId" = $1 AND status IN ('active', 'pending_setup')`
- Query Subscription: `SELECT "subscriptionType" as "productName", "subscriptionType" as type, status FROM "Subscription" s JOIN "TenantUser" tu ON s."userId" = tu."userId" WHERE tu."tenantId" = $1 AND s.status = 'active'`
  - NOTE: Subscription has userId not tenantId — join through TenantUser table. If TenantUser doesn't exist, fall back to matching via clientId on ServiceInstance.
  - Deduplicate by productName (ServiceInstance takes priority over Subscription for same product)
- Return combined array

Export `function getProductModuleHints(productName: string): string`:
- Map product names to onboarding module descriptions:
  - VideoForge / Winner Studio / Character-in-a-Box -> "I'll help create your AI brand character through a fun questionnaire, then generate a reveal video"
  - SocialHub / Buzz -> "I'll collect your social media preferences and set up your automated content pipeline"
  - FB Bot -> "I'll configure your Facebook Marketplace automation"
  - FrontDesk Voice AI -> "I'll set up your AI receptionist with your business hours and greeting preferences"
  - Lead Pages -> "I'll gather your brand assets and content for your lead landing pages"
  - Maps/SEO -> "I'll collect your competitor info and set up Google Maps monitoring"
  - Custom Solutions -> "I'll work with you on your custom project requirements"
  - Default -> "I'll help you get started with {productName}"

Export `async function assembleProductPrompt(tenantId: string): Promise<{ prompt: string; products: ProductInfo[] }>`:
- Call fetchTenantProducts(tenantId)
- Build system prompt sections:
  1. Role: "You are SuperSeller AI, a business onboarding specialist."
  2. Product awareness: "This customer has the following active products/services:" + bullet list of products
  3. Module hints: For each product, include getProductModuleHints() so agent knows what flows to activate
  4. Rules: WhatsApp-friendly format, never share pricing/internal details, one topic at a time, confirm before proceeding
  5. Language: "Match the customer's language. If they write in Hebrew, respond in Hebrew."
- Return { prompt, products }

For tests: Mock the `query` function from `../../db/client` to return controlled data. Test prompt assembly logic without DB.

Export `async function bootstrapOnboardingGroup(tenantId: string, clientPhone: string): Promise<OnboardingBootstrapResult>`:

Step-by-step (follow the pattern in character-pipeline/group-bootstrap.ts):

1. **Fetch tenant**: `SELECT id, name, slug FROM "Tenant" WHERE id = $1` — throw if not found
2. **Fetch brand**: `SELECT id, "logoUrl" as logo_url, "primaryColor" as primary_color, "accentColor" as accent_color, tagline FROM "Brand" WHERE "tenantId" = $1`
   - NOTE: Prisma schema calls it `Brand` not `TenantBrand`. The old code used `"TenantBrand"` — check which table name actually exists in DB. The Prisma model name is `Brand` so the table is `"Brand"`.
3. **Assemble product prompt**: Call `assembleProductPrompt(tenantId)` from prompt-assembler.ts
4. **Create PipelineRun**: pipelineType = "customer-onboarding", inputJson = { clientPhone, tenantName, products: products.map(p => p.productName), step: "group-bootstrap" }
5. **Create WhatsApp group**:
   - Name: `"{tenant.name} — SuperSeller AI"` (per UGRP-02)
   - Use `config.wahaSessions.biz` as wahaTarget session
   - Throw if createGroup returns null
6. **Set group icon**: If brand?.logo_url exists, call setGroupIcon(groupId, brand.logo_url, wahaTarget) (UGRP-04). Log warning if fails, don't throw.
7. **Set group description**: Include customer name + product list (UGRP-05):
   ```
   {tenant.name} — Powered by SuperSeller AI

   Active services:
   {products.map(p => `- ${p.productName}`).join('\n')}
   ```
8. **Register AI agent**: Call registerGroup with:
   - groupId, tenantId
   - agentName: "SuperSeller AI"
   - agentRole: "Universal business onboarding assistant"
   - systemPromptAdditions: the assembled product prompt
   - allowedPhones: [] (allow all)
   - language: undefined (match user's language)
9. **Send welcome message** (PAGENT-04):
   ```
   *Welcome to SuperSeller AI, {tenant.name}!*

   I'm your dedicated AI assistant. Here's what I'll help you with:

   {For each product: "- *{productName}*: {getProductModuleHints(productName)}"}

   Let's get started! I'll walk you through setup one step at a time.

   What would you like to begin with?
   ```
10. **Update PipelineRun**: status still "running" (onboarding continues), outputJson = { groupId, step: "group-bootstrap-complete", products }
11. **Return** OnboardingBootstrapResult

Error handling: Wrap steps 5-10 in try/catch. On failure, update PipelineRun with status="failed" + errorMessage. Re-throw.

Import from prompt-assembler.ts (co-located), waha-client.ts, group-agent.ts, pipeline-run.ts, config, logger — same pattern as character-pipeline version.
