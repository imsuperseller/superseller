- caption contains "team" or "staff" -> "team_photo"

- mediaType starts with "video" -> "video"
- mediaType starts with "application" -> "document"

Implement `social-setup.ts` as an OnboardingModule:

State machine phases: `intro` -> `asking_platforms` -> `asking_frequency` -> `asking_style` -> `confirming` -> `complete`

- `shouldActivate(products)`: Check for SocialHub, Buzz (case-insensitive).

- `getIntroMessage(tenantName)`: Explain what SocialHub does:
  "Now let's set up your social media presence! *SocialHub* will automatically create and post content for your business.

  Here's what it does:
  - Generates professional posts with your brand voice
  - Posts on schedule to your chosen platforms
  - Tracks engagement and optimizes content

  Which platforms do you want to post on? (e.g., Instagram, Facebook, LinkedIn)"

  Transition to 'asking_platforms' phase.

- `handleMessage()` per phase:
  - `asking_platforms`: Parse platform names from text (Instagram, Facebook, LinkedIn, Twitter/X, TikTok). Store in collectedData.platforms. Ask "How often do you want to post? (e.g., daily, 3x/week, weekly)"

`updateServiceConfig(tenantId, productName, configPatch)` helper (shared, put in module-state.ts or a shared utils):
- SELECT id, configuration FROM ServiceInstance WHERE tenantId AND productName IN ('SocialHub', 'Buzz') AND status IN ('active', 'pending_setup')
- Merge configPatch into existing configuration JSON
- UPDATE ServiceInstance SET configuration = merged

IMPORTANT: Do NOT collect actual passwords/tokens. Only collect preferences (platforms, frequency, style). Per research: "Do NOT store actual passwords in ServiceInstance.configuration."
