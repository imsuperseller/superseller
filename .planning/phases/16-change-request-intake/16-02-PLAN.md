**File 1: `apps/worker/src/services/onboarding/change-request-handler.ts`** (NEW)

Create the orchestrator with two exported functions:

1. **`handleChangeRequest(params)`** — Main entry point called from character-video-gen delivered phase.
   Params: `{ groupId: string, tenantId: string, messageBody: string, state: ModuleState }`
   Returns: `Promise<ModuleHandleResult>`

   Flow:
   a. Call `getLatestCharacterBible(tenantId)` to get current bible with scenario names
   b. Extract scenarioNames from `bible.metadata?.scenario_names ?? []` and sceneCount from `bible.metadata?.scene_count ?? 5`
   c. Call `classifyChangeRequest(messageBody, scenarioNames, sceneCount)`
   d. Create a change_requests row immediately via `createChangeRequest({ groupId, tenantId, messageBody, intent: result.intent, scope: result.intent === "scene-change" ? "scene" : result.intent === "character-change" ? "character" : null, sceneNumber: result.sceneNumber, changeSummary: result.changeSummary })`
   e. Handle each intent:
      - **`scene-change`**: If `result.sceneNumber` is null, treat as ambiguous (ask "Which scene?"). Otherwise, compute cost via `estimateChangeCost("scene-change", 1)`, update status to `awaiting-confirmation`, send poll via `sendPoll(groupId, "Regenerating scene ${sceneNumber} will cost $${(costCents/100).toFixed(2)}. Proceed?", ["Yes", "No"])`, store poll message ID via `updateChangeRequestStatus(crId, "awaiting-confirmation", { estimatedCostCents: costCents, pollMessageId })`. Return `{ handled: true }`.
      - **`character-change`**: Compute cost via `estimateChangeCost("character-change", sceneCount)`, update status to `awaiting-confirmation`, send poll with multi-scene cost, store poll message ID. Return `{ handled: true }`.
      - **`positive-feedback`**: Send `sendText(groupId, "Thank you! Glad you like it. If you'd like any changes, just describe what you'd like different.")`. Return `{ handled: true }`. Do NOT transition phase or close window.
      - **`unrelated`**: Return `{ handled: false }` so ClaudeClaw handles it normally.
      - **`ambiguous`**: Send `sendText(groupId, result.clarifyingQuestion ?? "Could you describe what you'd like changed?")`. Return `{ handled: true }`.

   Error handling: Wrap entire function in try/catch. On error, log with logger.error, call sendAdminAlert, return `{ handled: true, response: "Something went wrong processing your request. Our team will follow up." }`.

2. **`handleChangeRequestPollVote(params)`** — Called from claudeclaw.worker.ts when poll vote is for a change request.
   Params: `{ groupId: string, tenantId: string, selectedOption: string, changeRequestId: string }`
   Returns: `Promise<void>`

   Flow:
   a. If `selectedOption` matches "Yes" (case-insensitive):
      - `updateChangeRequestStatus(changeRequestId, "confirmed")`
      - `sendText(groupId, "Change request confirmed! We'll start working on it shortly.")` (Phase 17 picks up from confirmed status)
      - Log: `logger.info({ msg: "Change request confirmed", changeRequestId, groupId })`
      - NOTE: Do NOT dispatch to any BullMQ queue — Phase 17 will add the character-regen queue dispatch
   b. If "No":
      - `updateChangeRequestStatus(changeRequestId, "rejected")`
      - `sendText(groupId, "No problem! Your current video stays as-is. Send another message anytime if you'd like changes.")`
   c. Any other option: log warning, ignore

Import from Plan 01 services: `classifyChangeRequest`, `getLatestCharacterBible`, `createChangeRequest`, `updateChangeRequestStatus`, `estimateChangeCost`, `SORA_COST_PER_SCENE_CENTS`. Import `sendText`, `sendPoll` from `../waha-client`. Import `sendAdminAlert` from `../admin-alerts`. Import `logger` from `../../utils/logger`.

```typescript
if (groupCfg?.tenantId && job.data.isPollVote && job.data.pollOption) {
    // Check if this is a change-request confirmation poll
    const { getPendingChangeRequest } = await import("../../services/onboarding/change-request-intake");
    const pendingRequest = await getPendingChangeRequest(chatId);
    if (pendingRequest) {
        const { handleChangeRequestPollVote } = await import("../../services/onboarding/change-request-handler");
        await handleChangeRequestPollVote({
            groupId: chatId,
            tenantId: groupCfg.tenantId,
            selectedOption: job.data.pollOption,
            changeRequestId: pendingRequest.id,
        });
        return { handled: "change-request-poll-vote", isGroup: true };
    }
    // Fall through to module selection / pipeline poll
    const { handlePipelineEvent } = await import("./onboarding.worker");
    await handlePipelineEvent({
        type: "poll-vote",
        groupId: chatId,
        tenantId: groupCfg.tenantId,
        selectedLabel: job.data.pollOption,
    });
    return { handled: "pipeline-poll-vote", isGroup: true };
}
```
