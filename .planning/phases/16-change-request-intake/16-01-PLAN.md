**File 1: `apps/worker/src/services/onboarding/intent-classifier.ts`**

1. **`ChangeRequestClassification` interface** (exported):
   ```typescript
   export interface ChangeRequestClassification {
       intent: "scene-change" | "character-change" | "positive-feedback" | "unrelated" | "ambiguous";
       sceneNumber: number | null;
       changeSummary: string;
       clarifyingQuestion?: string;
   }
   ```

2. **`classifyChangeRequest(messageBody, scenarioNames, sceneCount)`** — async, exported. Makes a single Claude API call using the same `fetch()` pattern as `character-bible-generator.ts`:
   - URL: `https://api.anthropic.com/v1/messages`
   - Model: `claude-sonnet-4-20250514`
   - Headers: `x-api-key: process.env.ANTHROPIC_API_KEY`, `anthropic-version: 2023-06-01`, `content-type: application/json`
   - System prompt builds scene list from scenarioNames (if available) with fallback to ordinal-only matching when scenarioNames is empty/null. When scenarioNames is empty, Claude prompt explicitly says "match by ordinal only (first=1, second=2, etc.)" and for description-based references returns `intent: "ambiguous"` with `clarifyingQuestion: "Which scene number would you like to change? (1-5)"`.
   - User message: the customer's messageBody
   - Response parsing: Extract JSON from Claude's text response using `JSON.parse()` with try/catch. If parsing fails, return `{ intent: "ambiguous", sceneNumber: null, changeSummary: "", clarifyingQuestion: "Could you describe what you'd like changed?" }`.
   - After successful classification, call `trackExpense({ service: "anthropic", operation: "intent-classification", estimatedCost: COST_RATES.anthropic.haiku_message })` — use haiku rate since this is a small classification call. Import from `../expense-tracker`.
   - On Claude API failure (non-200 response or network error), log with `logger.error`, call `sendAdminAlert` with the error, and return the ambiguous fallback.

3. **`buildClassificationPrompt(messageBody, scenarioNames, sceneCount)`** — internal function (not exported). Builds the system prompt with scene list and classification rules. Include ordinal mapping table in the prompt (first=1, second=2, third=3, fourth=4, fifth=5). Include the 5 intent categories with clear definitions.

2. **`getLatestCharacterBible(tenantId)`** — async, exported. Queries:
   ```sql
   SELECT id, "tenantId", name, "personaDescription", "visualStyle", "soraHandle", metadata, version, "changeDelta", "createdAt"
   FROM "CharacterBible"
   WHERE "tenantId" = $1
   ORDER BY "createdAt" DESC
   LIMIT 1
   ```
   Returns typed object or null. Parse metadata and changeDelta from JSON if they come as strings.

3. **`createCharacterBibleVersion(tenantId, updatedFields, changeDelta)`** — async, exported.
   - Calls `getLatestCharacterBible(tenantId)` to get current version
   - If no current bible exists, return null

   - Merges: `{ ...current, ...updatedFields }` for all CharacterBibleFields
   - INSERTs new row with gen_random_uuid()::text as id, incremented version, changeDelta as JSONB
   - Returns the new row's id (from RETURNING id) or null
   - NEVER updates the existing row

Both files use `import { query, queryOne } from "../../db/client"`, `import { logger } from "../../utils/logger"`.
