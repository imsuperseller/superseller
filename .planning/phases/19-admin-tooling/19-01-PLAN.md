Create three API route files following the established admin auth pattern (verifySession + role check).

**Route 1: GET /api/admin/character-versions/[tenantId]/route.ts**
- Auth: checkAuth() pattern from clients/route.ts
- Query CharacterBible versions for tenantId using `prisma.$queryRaw` (to get changeDelta reliably and LEFT JOIN with change_requests in one query):
  ```sql
  SELECT cb.id, cb."tenantId", cb.name, cb."visualStyle", cb."soraHandle",
         cb."personaDescription", cb.version, cb."changeDelta", cb."createdAt",
         cr.intent, cr.scope, cr.scene_number, cr.estimated_cost_cents, cr.status as cr_status, cr.change_summary
  FROM "CharacterBible" cb
  LEFT JOIN change_requests cr ON cr.character_bible_version_id = cb.id
  WHERE cb."tenantId" = ${tenantId}
  ORDER BY cb.version DESC
  ```
- Return `{ versions: [...] }` with 200

**Route 2: POST /api/admin/character-versions/[tenantId]/rollback/route.ts**
- Auth: same checkAuth() pattern
- Body: `{ targetVersion: number }` — validate targetVersion is a positive integer
- Fetch target version row: `SELECT * FROM "CharacterBible" WHERE "tenantId" = $tenantId AND version = $targetVersion LIMIT 1`
- If not found, return 404 `{ error: "Version not found" }`
- Insert new row with atomic version increment (prevents Pitfall 4 — version collision):
  ```sql
  INSERT INTO "CharacterBible" (id, "tenantId", name, "personaDescription", "visualStyle", "soraHandle", metadata, version, "changeDelta", "createdAt", "updatedAt")
  VALUES (
    gen_random_uuid()::text,
    ${tenantId},
    ${target.name},
    ${target.personaDescription},
    ${target.visualStyle},
    ${target.soraHandle},
    ${target.metadata}::jsonb,
    (SELECT COALESCE(MAX(version), 0) + 1 FROM "CharacterBible" WHERE "tenantId" = ${tenantId}),
    ${JSON.stringify({ rollback: { fromVersion: currentVersion, toVersion: targetVersion, rolledBackBy: 'admin' } })}::jsonb,
    NOW(),
    NOW()
  )
  RETURNING id, version
  ```
- Return `{ success: true, newVersion: { id, version } }` with 200

**Route 3: GET /api/admin/change-requests/[tenantId]/route.ts**
- Auth: same checkAuth() pattern
- Read optional `?status=` query param from URL searchParams
- Query change_requests via `prisma.$queryRaw`:
  - If status filter provided, use two separate query branches (Prisma tagged templates don't support conditional fragments — see Research Pitfall note)
  - Without filter: `SELECT * FROM change_requests WHERE tenant_id = ${tenantId} ORDER BY created_at DESC`
  - With filter: `SELECT * FROM change_requests WHERE tenant_id = ${tenantId} AND status = ${statusFilter} ORDER BY created_at DESC`
- Compute totalCostCents: sum of estimated_cost_cents where status = 'completed' or 'approved'
- Return `{ changeRequests: [...], totalCostCents: number }` with 200

All routes: `export const dynamic = 'force-dynamic';` at top. Import NextRequest, NextResponse from 'next/server', verifySession from '@/lib/auth', prisma from '@/lib/prisma'.

**Pitfall guards:**
- Use quoted camelCase column names for CharacterBible queries ("tenantId", "changeDelta", etc.)
- Use bare snake_case for change_requests queries (tenant_id, estimated_cost_cents, etc.)
- Accept tenantId as UUID (not slug) — the URL param IS the UUID
