  <action>
Create a single client component `CharacterHistoryTab.tsx` with two inner sub-views toggled by local state: "versions" (default) and "audit-log".

**Component structure:**
- `'use client'` at top
- State: `selectedTenantId`, `versions[]`, `changeRequests[]`, `totalCostCents`, `loading`, `rollbackTarget` (number|null for modal), `subView` ('versions'|'audit'), `statusFilter` (string|null)
- On mount: fetch tenant list from `/api/admin/tenants`
- On tenant selection: fetch both `/api/admin/character-versions/${tenantId}` and `/api/admin/change-requests/${tenantId}` in parallel

**Tenant selector:**
- Dropdown at top of component showing tenant name + slug
- Default: no tenant selected, show "Select a customer to view character history"

**Sub-view toggle:**
- Two buttons/tabs: "Version Timeline" | "Change Log"
- Style: same as existing admin tab patterns (uppercase tracking-widest)

**Versions sub-view (ADMIN-02):**
- Vertical timeline layout, most recent version at top
- Each version card shows:
  - Version number badge (e.g., "v3") + date formatted as locale string
  - If changeDelta is a rollback: show "Rolled back to v{toVersion}" with a special badge
  - If changeDelta has field changes: show old->new for each changed field in a small diff display
  - If changeDelta is null (initial version): show "Initial version"
  - Cost: if estimated_cost_cents from joined CR data exists, show as "$X.XX" (divide by 100). If null, show "---"
  - Status badge: map cr_status to colors (completed=green, approved=blue, failed=red, null=gray "initial")
  - Rollback button: appears on every version EXCEPT the current (first/latest) one. onClick sets rollbackTarget.
- Empty state: "No character changes recorded for this customer"

**Rollback confirmation modal (ADMIN-03):**
- Triggered when rollbackTarget !== null
- Dark overlay modal (use pattern from AuditManagement.tsx --- fixed inset-0, bg-black/60, centered card)
- Text: "Roll back to version {N}? This creates a new version with those settings. Future scene generations will use these character settings."
- Two buttons: "Confirm Rollback" (destructive variant) and "Cancel" (outline variant)
- On confirm: POST to `/api/admin/character-versions/${tenantId}/rollback` with `{ targetVersion: rollbackTarget }`
- On success: re-fetch versions, close modal
- On error: show error message, keep modal open

**Audit log sub-view (ADMIN-01):**
- Table with columns: Date, Intent, Summary, Scene(s), Status, Cost (cents)
- Status filter dropdown: All, received, confirmed, completed, failed, pending-admin-approval, admin-denied
- When filter changes: re-fetch `/api/admin/change-requests/${tenantId}?status=${filter}` (omit param for "All")
- Sort by date descending (API already returns this order)
- Total cost row at bottom: "Total: {totalCostCents} cents" --- bold, right-aligned
- Status badges with colors: received=gray, confirmed=blue, completed=green, failed=red, pending-admin-approval=yellow, admin-denied=red
- Empty state: "No change requests for this customer"

**Styling:**
- Follow existing admin dark theme: bg-slate-900/bg-black borders, text-white/text-slate-400, cyan accents
- Use Tailwind classes consistent with existing admin components
- Import Lucide icons: History, RotateCcw, Clock, Filter
