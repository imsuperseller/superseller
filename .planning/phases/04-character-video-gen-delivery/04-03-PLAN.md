  <action>
Extend the character-video-gen module (from Plan 01) to add composition, upload, and delivery after scene generation completes.

**Add new state machine phases** after `awaiting-composition`:

**`composing` phase:**
1. Read `collectedData.sceneUrls` (R2 URLs from Plan 01) and `collectedData.sceneLabels`
2. Download scene clips from R2 to local `/tmp/character-video/{tenantId}/` if not already local (they should be local from generation, but handle re-entry gracefully)
3. Fetch tenant Brand data: `SELECT "accentColor", "logoUrl" FROM "Brand" WHERE "tenantId" = $1 LIMIT 1` — use defaults if no Brand row
4. Fetch CharacterBible for name/tagline: reuse the query from the generation phase
5. Build `CharacterRevealProps`:
   - `characterName`: CharacterBible.name
   - `businessName`: fetch from Tenant table: `SELECT name FROM "Tenant" WHERE id = $1`
   - `tagline`: CharacterBible.personaDescription truncated to 50 chars + "..."
   - `accentColor`: from Brand or default "#C9A96E"
   - `bgColor`: "#0A0A0A"
   - `logoUrl`: from Brand (optional)
   - `sceneClips`: local file paths of the 5 scene clips
   - `sceneLabels`: from collectedData (["Studio Portrait", scenario1_name, scenario2_name, scenario3_name, "Artistic Closer"])
6. Call `renderComposition({ compositionId: "CharacterReveal-16x9", inputProps, outputPath: "/tmp/character-video/{tenantId}/reveal.mp4" })`
7. Store `collectedData.revealPath` = outputPath
8. Transition to `uploading`

**`uploading` phase:**
1. Upload `/tmp/character-video/{tenantId}/reveal.mp4` to R2 at key `character-videos/{tenantId}/reveal-{timestamp}.mp4`
2. Pass `assetInfo: { tenantId, type: "video", filename: "character-reveal.mp4", description: "AI Character Reveal Video" }` for TenantAsset registration
3. Store `collectedData.revealUrl` = R2 public URL
4. Transition to `delivering`

**`delivering` phase:**
1. Build summary caption:
   - Line 1: "{characterName} - Your AI Character Reveal"
   - Line 2: (blank)
   - Line 3: "5 scenes showcasing your brand character in action:"
   - Lines 4-8: "- {sceneLabel}" for each scene
   - Line 9: (blank)
   - Line 10 (conditional): If tenant has credit-based billing (check ServiceInstance or Subscription for billing_model = "credit"), append "Credits used: {totalCost}". Otherwise omit.
   - Final line: "Your character is ready! Reply here if you'd like any changes."
2. Call `sendVideo(groupId, revealUrl, caption)` — groupId from state
3. Update PipelineRun: `updatePipelineRun(pipelineRunId, { status: "completed", deliveredVia: "whatsapp", deliveredAt: new Date(), costCents: totalCostCents, outputJson: { revealUrl, sceneCount: sceneClips.length } })`
4. Transition to `complete` with `completed: true`

**`complete` phase:**
- Respond with "Your character reveal has been delivered! Let me know if you'd like anything adjusted."
- Return `{ handled: true, completed: true }`

**Error handling:**
- If renderComposition fails: retry once. If still fails, update PipelineRun with `status: "failed"`, errorMessage, send text message to group: "There was an issue composing your video. Our team has been notified."
- If R2 upload fails: same pattern — retry once, fail gracefully
- If WAHA sendVideo fails: log error, update PipelineRun with `deliveredVia: "failed"`, respond in group with text fallback containing R2 URL

**Billing model detection** for cost visibility:
```sql
SELECT configuration->>'billing_model' as billing_model
FROM "ServiceInstance"
WHERE "tenantId" = $1 AND "productType" ILIKE '%video%'
LIMIT 1
```
If `billing_model = "credit"`, show cost. If null or `"flat"` or `"package"`, omit cost line. Default to omitting cost if query fails.

**Cleanup:** After successful delivery, delete `/tmp/character-video/{tenantId}/` directory to free disk space.
