### Integration Issues (Non-Blocking)

**INT-01: Checkout navigation gap**
- Subscriptions page (`ClientPage.tsx`) routes to old `/api/checkout?flowType=subscription`
- New `/checkout/[product]` pages with phone collection are unreachable from navigation
- Impact: Customers can only reach new flow via direct URL — not a requirement blocker but a UX gap

**INT-02: PayPal custom_id key mismatch**
- `CheckoutForm.tsx` stores `{phone, bn, svc}` in custom_id
- `paypal/route.ts` reads `customMetadata.productName` (doesn't exist) → defaults to 'SuperSeller AI'
- Impact: Onboarding fires correctly but with generic product name

**INT-03: Stripe duplicate ServiceInstance**
- Stripe webhook step 4 creates ServiceInstance AND `onboardNewCustomer()` creates another
- Impact: Two ServiceInstance records per Stripe checkout when phone is present
