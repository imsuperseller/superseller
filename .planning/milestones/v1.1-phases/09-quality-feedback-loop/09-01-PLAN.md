**Part A — Extend trackExpense calls with model_id/provider:**

**Part B — Create content_entry at job completion with generation_meta and performanceScore:**

At the canonical job completion point (after `UPDATE video_jobs SET status = 'complete'`, around line 1146), add a new block that:

1. Collects generation metadata from the completed job:
   - `model_id`: from the clips table or the modelUsed variable (use the dominant model — most clips use the same model)
   - `provider`: 'kie' or 'fal' based on model prefix
   - `prompt_key`: from the prompt store if available, otherwise 'default'
   - `prompt_version`: '1.0' for v1.1 (no version tracking yet)
   - `generation_cost_usd`: from totalCost variable already computed
   - `duration_sec`: from masterDuration variable already computed
   - `shot_type`: from the job's shot type or 'cinematic' default
   - `external_job_id`: jobId
   - `image_url_used`: first frame URL if available

2. Calculates performanceScore as a composite 0.0-1.0:
   ```typescript
   const SUCCESS_WEIGHT = 0.34;
   const COST_WEIGHT = 0.33;
   const DURATION_WEIGHT = 0.33;

   const successScore = 1.0; // We're in the completion path, so status = completed
   const expectedCost = clips.length * (config.video.klingMode === 'pro' ? 0.10 : 0.03);
   const costScore = Math.min(expectedCost / Math.max(totalCost, 0.01), 1.0);
   const requestedDuration = clips.reduce((sum, c) => sum + (c.duration_seconds || 5), 0);
   const durationScore = Math.min(requestedDuration / Math.max(masterDuration, 1), 1.0);
   const performanceScore = SUCCESS_WEIGHT * successScore + COST_WEIGHT * costScore + DURATION_WEIGHT * durationScore;
   ```

3. Inserts a content_entries row with generation_meta and performanceScore:
   ```typescript
   await query(
     `INSERT INTO content_entries (id, tenant_id, type, status, media_url, performance_score, generation_meta, created_at, updated_at)
      VALUES (gen_random_uuid(), $1, 'reel', 'published', $2, $3, $4, NOW(), NOW())
      ON CONFLICT DO NOTHING`,
     [tenantId || userId, masterUrl, performanceScore, JSON.stringify(generationMeta)]
   );
   ```

   The tenantId should come from the job data. If the job doesn't have a tenantId, use userId as fallback. The ON CONFLICT DO NOTHING makes this idempotent (safe on retry).

4. Gate this write: only execute if this is a NEW completion (not the idempotent early-return at line 52-57). The existing early-return check already handles this — the content_entry insert runs after the main completion UPDATE, so it will only run on first completion.

**Important:** The content_entry INSERT must be wrapped in try/catch and logged but NEVER throw — same pattern as trackExpense. Don't fail the pipeline over metadata tracking.
